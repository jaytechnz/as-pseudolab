Pseudocode Reference

Cambridge AS & A Level Computer Science 9618 pseudocode conventions.

Comments 
// This is a comment

Comments are ignored by the interpreter. Use them to explain your code.

Data Types
INTEGERWhole numbers, e.g. 42, -7
REALDecimals, e.g. 3.14
CHARSingle character, e.g. 'A'
STRINGText, e.g. "Hello"
BOOLEANTRUE or FALSE
DATEe.g. 28/03/26
Variables & Constants

Declare a variable:

DECLARE Count : INTEGER
DECLARE Score : REAL
DECLARE UserName : STRING
DECLARE IsValid : BOOLEAN

Declare a constant:

CONSTANT PI = 3.14159
CONSTANT MAX_SIZE = 100

Assignment (use the ← arrow):

Count ← 0
Score ← Score + 1

Variable, array, procedure and function names begin with a capital letter. Constants are full caps with SNAKE_CASE.

Input & Output 
INPUT UserName

OUTPUT "Text"
OUTPUT Score
OUTPUT "Score = ", Score
OUTPUT "Sum = ", NumA + NumB

OUTPUT can combine multiple values separated by commas.

Operators

Arithmetic:

+Addition
-Subtraction
*Multiplication
/Division (real result)
DIVInteger division
MODRemainder
^Power / exponent

Comparison:

=Equal to
<>Not equal to
<Less than
>Greater than
<=Less than or equal
>=Greater than or equal

Logical:

ANDBoth conditions true
OREither condition true
NOTNegates a condition

String:

&Concatenation: "Hi" & UserName
Selection — IF 
IF condition
   THEN
      statements
ENDIF

IF condition
   THEN
      statements
   ELSE
      statements
ENDIF

Example:

IF Score >= 50
   THEN
      OUTPUT "Pass"
   ELSE
      OUTPUT "Fail"
ENDIF

Nested IF statements are allowed. Always close with ENDIF.

Selection — CASE 
CASE OF Variable
   value1 : statement
   value2 : statement
   OTHERWISE : statement
ENDCASE

Example:

CASE OF Grade
   "A" : OUTPUT "Excellent"
   "B" : OUTPUT "Good"
   "C" : OUTPUT "Average"
   OTHERWISE : OUTPUT "Below average"
ENDCASE

OTHERWISE is optional but good practice. Use for simple value matching.

Iteration — FOR 
FOR Counter ← start TO finish
   statements
NEXT Counter

FOR Counter ← start TO finish STEP n
   statements
NEXT Counter

Example — count up:

FOR Counter ← 1 TO 5
   OUTPUT Counter
NEXT Counter

Example — count down in twos:

FOR Counter ← 10 TO 0 STEP -2
   OUTPUT Counter
NEXT Counter

Use FOR when the number of iterations is known in advance.

Iteration — WHILE 
WHILE condition
   statements
ENDWHILE

Example:

DECLARE Count : INTEGER
Count ← 1
WHILE Count <= 10
   OUTPUT Count
   Count ← Count + 1
ENDWHILE

The condition is checked before each iteration. The loop body may never execute if the condition starts false. Note: no DO keyword in 9618 pseudocode.

Iteration — REPEAT 
REPEAT
   statements
UNTIL condition

Example — input validation:

REPEAT
   INPUT Age
UNTIL Age >= 0 AND Age <= 120

The condition is checked after each iteration. The loop body always runs at least once.

Arrays

1D array — declare and use:

DECLARE Scores : ARRAY[1:10] OF INTEGER
Scores[1] ← 85
Scores[2] ← 92
OUTPUT Scores[1]

2D array — declare and use:

DECLARE Grid : ARRAY[1:3, 1:3] OF INTEGER
Grid[1,1] ← 0
Grid[2,3] ← 7
OUTPUT Grid[2,3]

Loop through an array:

DECLARE Total : INTEGER
Total ← 0
FOR Counter ← 1 TO 10
   Total ← Total + Scores[Counter]
NEXT Counter

Array indices start at the lower bound you declare (often 1, sometimes 0).

Records (User-defined Types)

Define a record type:

TYPE StudentRecord
   DECLARE LastName : STRING
   DECLARE FirstName : STRING
   DECLARE DateOfBirth : DATE
   DECLARE YearGroup : INTEGER
   DECLARE FormGroup : CHAR
ENDTYPE

Declare variables of that type:

DECLARE Pupil1 : StudentRecord
DECLARE Pupil2 : StudentRecord
DECLARE Form : ARRAY[1:30] OF StudentRecord

Access fields with dot notation:

Pupil1.LastName ← "Johnson"
Pupil1.FirstName ← "Leroy"
Pupil1.YearGroup ← 6
Pupil1.FormGroup ← 'A'

// Copy one record to another
Pupil2 ← Pupil1

// Access a field in an array of records
Form[1].YearGroup ← Form[1].YearGroup + 1

Pass a record to a procedure:

PROCEDURE PrintStudent(S : StudentRecord)
   OUTPUT S.FirstName, " ", S.LastName
ENDPROCEDURE

CALL PrintStudent(Pupil1)

Record types are defined with TYPE...ENDTYPE. Fields are accessed using dot notation. Records can be stored in arrays and passed as parameters.

Procedures & Parameter Passing

No parameters:

PROCEDURE ShowMenu()
   OUTPUT "1. Start"
   OUTPUT "2. Quit"
ENDPROCEDURE

CALL ShowMenu()

Pass by value (BYVAL — default):

PROCEDURE PrintDouble(BYVAL Num : INTEGER)
   Num ← Num * 2      // only changes the local copy
   OUTPUT Num
ENDPROCEDURE

DECLARE Score : INTEGER
Score ← 5
CALL PrintDouble(Score)
OUTPUT Score   // still 5 — original unchanged

Pass by reference (BYREF — modifies original):

PROCEDURE Swap(BYREF X : INTEGER, Y : INTEGER)
   DECLARE Temp : INTEGER
   Temp ← X
   X ← Y
   Y ← Temp
ENDPROCEDURE

DECLARE A : INTEGER
DECLARE B : INTEGER
A ← 10
B ← 20
CALL Swap(A, B)
OUTPUT A, " ", B   // 20 10

Multiple parameter types:

PROCEDURE UpdateRecord(BYREF S : StudentRecord,
                       BYVAL NewYear : INTEGER)
   S.YearGroup ← NewYear
ENDPROCEDURE

If no keyword is given, BYVAL is assumed. BYREF is needed when the procedure must change the caller's variable. The BYREF/BYVAL keyword applies to all following parameters until another keyword appears.

Functions

Define and use:

FUNCTION Max(Number1 : INTEGER, Number2 : INTEGER)
            RETURNS INTEGER
   IF Number1 > Number2 THEN
      RETURN Number1
   ELSE
      RETURN Number2
   ENDIF
ENDFUNCTION

// Call inside an expression — no CALL keyword
OUTPUT "Penalty Fine = ", Max(10, Distance * 2)

DECLARE Best : INTEGER
Best ← Max(Score1, Score2)

No-parameter function:

FUNCTION GetGrade() RETURNS CHAR
   DECLARE Mark : INTEGER
   INPUT Mark
   IF Mark >= 70 THEN
      RETURN 'A'
   ELSE
      RETURN 'B'
   ENDIF
ENDFUNCTION

OUTPUT "Grade: ", GetGrade()

String-returning function:

FUNCTION Classify(Score : INTEGER) RETURNS STRING
   IF Score >= 50 THEN
      RETURN "Pass"
   ELSE
      RETURN "Fail"
   ENDIF
ENDFUNCTION

Functions return a single value with RETURN. Use them inside expressions — never with CALL. Parameters should not be passed BYREF to a function.

String Operations
LENGTH(s)Returns INTEGER — number of characters in string s
MID(s, x, y)Returns STRING — y characters from position x (1-indexed)
RIGHT(s, x)Returns STRING — rightmost x characters of s
LCASE(c)Returns CHAR — lower-case equivalent of character c
UCASE(c)Returns CHAR — upper-case equivalent of character c

Examples:

DECLARE Word : STRING
Word ← "Cambridge"
OUTPUT LENGTH(Word)        // 9
OUTPUT MID(Word, 1, 3)     // Cam
OUTPUT MID(Word, 5, 6)     // bridge
OUTPUT RIGHT(Word, 5)      // ridge

LCASE / UCASE operate on a single CHAR:

DECLARE Ch : CHAR
Ch ← 'W'
OUTPUT LCASE(Ch)   // 'w'
OUTPUT UCASE('h')  // 'H'

Concatenation with & (joins strings):

DECLARE Full : STRING
Full ← "Summer" & " " & "Pudding"
OUTPUT Full   // Summer Pudding

String functions will always be provided in exams. LCASE/UCASE accept a CHAR, not a full STRING.

Type Conversion
STR_TO_NUM(s)Converts STRING s to a numeric value (INTEGER or REAL)
NUM_TO_STR(n)Converts a numeric value n to a STRING

Examples:

DECLARE AgeStr : STRING
DECLARE Age : INTEGER
AgeStr ← "17"
Age ← STR_TO_NUM(AgeStr)   // Age = 17
OUTPUT Age + 1              // 18

DECLARE Score : INTEGER
DECLARE ScoreStr : STRING
Score ← 95
ScoreStr ← NUM_TO_STR(Score)
OUTPUT "Score: " & ScoreStr  // Score: 95

Useful when reading CSV data from files:

// CSV line: "Alice,85"
// after parsing the second field as a string:
Mark ← STR_TO_NUM(Field2)

Type conversion is needed when data read from a file (always a STRING) must be used in arithmetic. STR_TO_NUM will cause an error if the string is not a valid number.

Numeric Functions
INT(x)Returns INTEGER — truncates (drops) the fractional part of REAL x
RAND(x)Returns REAL — random number in range 0 to x (exclusive)
x DIV yInteger division — whole number quotient
x MOD yRemainder after integer division

Examples:

OUTPUT INT(27.5415)  // 27  (not rounded — truncated)
OUTPUT INT(-3.9)     // -3
OUTPUT RAND(87)      // e.g. 35.43  (0 ≤ result < 87)
OUTPUT 17 DIV 5      // 3
OUTPUT 17 MOD 5      // 2

INT() truncates towards zero — it does not round. RAND(x) returns a REAL, not an INTEGER.

File Handling
OPENFILE f FOR READOpen for reading
OPENFILE f FOR WRITEOpen for writing (overwrites existing)
OPENFILE f FOR APPENDOpen and add to end
READFILE f, vRead next line into STRING variable v
WRITEFILE f, dataWrite data as a line
CLOSEFILE fClose the file
EOF(f)Returns TRUE if at end of file

Write to a file:

OPENFILE "data.txt" FOR WRITE
WRITEFILE "data.txt", "Alice,85,TRUE"
WRITEFILE "data.txt", "Bob,72,FALSE"
CLOSEFILE "data.txt"

Read from a file:

DECLARE Line : STRING
OPENFILE "data.txt" FOR READ
WHILE NOT EOF("data.txt")
   READFILE "data.txt", Line
   OUTPUT Line
ENDWHILE
CLOSEFILE "data.txt"

Comma-separated values (CSV) — reading fields:

// Each line is a STRING: "Alice,85,TRUE"
// Split fields by finding commas manually using MID and LENGTH,
// or use a helper function provided in the question.
DECLARE Line : STRING
DECLARE Name : STRING
DECLARE Score : INTEGER
DECLARE Pos1 : INTEGER
DECLARE Pos2 : INTEGER

OPENFILE "students.csv" FOR READ
WHILE NOT EOF("students.csv")
   READFILE "students.csv", Line
   // find first comma position
   Pos1 ← 1
   WHILE MID(Line, Pos1, 1) <> ","
      Pos1 ← Pos1 + 1
   ENDWHILE
   Name ← MID(Line, 1, Pos1 - 1)
   // remaining fields follow after Pos1
   OUTPUT Name
ENDWHILE
CLOSEFILE "students.csv"

CSV files store each record as a line of comma-separated values. READFILE reads one line at a time as a STRING. Parse fields using MID and LENGTH. In exams, a helper function to split fields will usually be provided.

Quick Syntax Checklist
  • Every IF needs ENDIF
  • Every WHILE needs ENDWHILE — no DO
  • Every FOR needs NEXT
  • Every REPEAT needs UNTIL
  • Every CASE OF needs ENDCASE
  • Every PROCEDURE needs ENDPROCEDURE
  • Every FUNCTION needs ENDFUNCTION
  • Every TYPE (record) needs ENDTYPE
  • Call procedures with CALL; call functions without it
  • Parameters are BYVAL by default; use BYREF to modify original
  • LCASE / UCASE take a CHAR, not a full string
  • INT(x) truncates; it does not round
  • Use ← for assignment, not =
  • Use = only for comparison (in conditions)
  • Declare variables before using them
  • Strings use double quotes "text"
  • Characters use single quotes 'A'
